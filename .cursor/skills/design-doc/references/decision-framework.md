# Decision Framework

Record material decisions using this structure:

```markdown
### DEC-001 — Decision title

- Status: Proposed | Approved | Rejected | Superseded
- Date:
- Owner:
- Sources:

**Decision**

State the chosen direction.

**Rationale**

Explain why it supports user, business, brand, and technical needs.

**Alternatives considered**

- Alternative A
- Alternative B

**Consequences**

- Positive consequence
- Cost, limitation, or dependency

**Validation**

Describe how the decision will be verified.
```

## Decisions that always require records

- Primary conversion destination
- Page structure and route strategy
- Font choice and licensing path
- Contact/form architecture
- CMS decision
- Analytics/consent stack
- Treatment of professional or health-related claims
- Use of testimonials
- Performance budget exceptions
- Any deviation from brand guidelines
