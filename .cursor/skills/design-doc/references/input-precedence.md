# Input Precedence and Conflict Resolution

Use this order when sources conflict:

1. Explicit current user instruction
2. Approved decision record
3. Approved project brief or business requirement
4. Legal, privacy, security, and professional constraints
5. Existing repository architecture and deployment constraints
6. Approved brand guidelines
7. Approved content inventory
8. Research evidence
9. Existing unapproved drafts
10. Agent recommendation

## Conflict handling

For each conflict:

1. Quote or summarize both positions.
2. Identify affected sections or requirements.
3. Explain the consequence of each option.
4. Mark the decision `[OPEN]` unless precedence resolves it.
5. Never delete the losing alternative from the decision history.

## Source integrity

- Do not treat file recency alone as approval.
- Prefer explicit status and decision records over timestamps.
- Do not assume copy is legally or professionally approved because it exists in the repository.
